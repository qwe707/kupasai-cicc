# 飞书协作空间结构

## 一、设计原则

- **飞书 = 过程**：讨论、决策、任务、表格、日历
- **GitHub = 资产**：代码、文档版本、可追溯的提交包
- 所有最终交付物在 GitHub 留底；飞书放工作版本与多人协作版

## 二、飞书知识库结构

```
CICC 2026 · 建交赛道
├── 00 项目总览
│   ├── 项目一页纸
│   ├── 比赛信息摘要（同步自 docs/00-overview/competition-brief.md）
│   └── 团队与分工
├── 01 选题与方案
│   ├── 选题判断表
│   ├── 问题定义
│   ├── 技术方案
│   └── 评估方案
├── 02 周计划与日推进
│   ├── 周计划（周一开）
│   ├── 日站会记录
│   └── 月度复盘
├── 03 决策记录
├── 04 路演与答辩
│   ├── 路演脚本
│   ├── 答辩 Q&A 预案
│   └── PPT 评审记录
└── 05 资料库
    ├── 行业调研
    ├── 对标分析
    └── 参考论文 / 文章
```

## 三、飞书多维表格

### 1. 赛题池（topic-pool）
| 字段 | 类型 |
| --- | --- |
| 赛题 | 文本 |
| 方向 | 单选（建设/交通） |
| 业务价值 | 数字 1-5 |
| 数据可得 | 数字 1-5 |
| 技术可行 | 数字 1-5 |
| 展示效果 | 数字 1-5 |
| 时间风险 | 数字 1-5 |
| 总分 | 公式 |
| 状态 | 单选（候选/已选/已弃） |

### 2. 任务看板（tasks）
| 字段 | 类型 |
| --- | --- |
| 任务 | 文本 |
| 阶段 | 单选（M0/M1/M2/M3/M4） |
| 负责人 | 人员 |
| 协作人 | 人员（多） |
| 状态 | 单选（待办/进行中/阻塞/完成） |
| 优先级 | 单选（P0/P1/P2） |
| 截止 | 日期 |
| 关联 GitHub Issue | URL |

### 3. 里程碑（milestones）
| 字段 | 类型 |
| --- | --- |
| 里程碑 | 单选（M0..M4） |
| 目标 | 文本 |
| 关键交付 | 多选 |
| 计划日期 | 日期 |
| 实际日期 | 日期 |
| 状态 | 单选 |

### 4. 风险清单（risks）
| 字段 | 类型 |
| --- | --- |
| 风险 | 文本 |
| 类型 | 单选（数据/技术/合规/时间/团队） |
| 严重度 | 数字 1-5 |
| 概率 | 数字 1-5 |
| 应对策略 | 文本 |
| 负责人 | 人员 |
| 状态 | 单选（监控/缓解中/已解决） |

### 5. 材料清单（assets）
| 字段 | 类型 |
| --- | --- |
| 名称 | 文本 |
| 类型 | 单选（文档/PPT/视频/截图/代码/数据） |
| 阶段 | 单选（报名/初赛/决赛） |
| 链接 | URL |
| 版本 | 文本 |
| 负责人 | 人员 |
| 状态 | 单选 |

## 四、飞书云文件夹

```
CICC 2026 建交/

```

## 五、与 GitHub 同步约定

| 飞书侧 | GitHub 侧 | 同步策略 |
| --- | --- | --- |
| 比赛信息摘要 | `docs/00-overview/competition-brief.md` | 单向：GitHub 为准 |
| 选题判断表 | `docs/00-overview/topic-selection.md` | 双向，决策后冻结 |
| 任务看板 | `.github/ISSUE_TEMPLATE` + Issues | 多维表格→Issue 编号 |
| 风险清单 | `docs/04-research/risks.md` | 周同步 |
| 路演 PPT 版本 | `assets/ppt/` 与 `deliverables/final-round/` | 冻结版进 deliverables |
| 视频素材 | 飞书云 + `assets/video/`（小样） | 大文件留飞书 |
```
