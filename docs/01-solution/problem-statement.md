# 问题陈述：城市轨道交通保护区安全管控

> 赛道：建交 ｜ 标签：安全韧性
> 赛题全名：基于多模态大模型赋能的城市轨道交通保护区安全管控场景
> 应用产物（官方要求）：**城市轨道交通保护区实施判图智能体**

---

## 1. 业务背景

轨道交通安全管控正面临"外部环境不可控 + 内部场景高动态"的双重挑战，
传统人工巡线 / 人工看片的管控手段已经触及效率天花板。

随着城市化进程加快，轨道沿线**保护区**内的违规施工、异物侵限风险持续激增。
保护区一般指轨道线路两侧 30~50 米的安全控制区域（含高架沿线、地面线侧、隧道地表投影、车辆段周边等），
该区域内禁止违规施工、堆载、违章建筑、违规吊装、违规倾倒等行为。

## 2. 核心痛点：盲区多 + 时效差

| 维度 | 现状 | 风险 |
|---|---|---|
| 巡线覆盖 | 人工巡线日均仅 5 公里 | 大量盲区未触达 |
| 物理盲区 | 高架桥下、封闭河道、隧道地表上方等肉眼难达 | 风险隐蔽不可见 |
| 无人机使用 | "飞手遥控 + 事后人工看片" | 海量影像堆积，无法实时判别 |
| 风险类型 | 挖掘机施工、堆土超高、异物入侵等 | 风险发现滞后，易演变为行车事故 |
| 决策链路 | 看片 → 上报 → 复核 → 派单 | 全人工串行，分钟级到小时级 |

简言之：**风险点在持续增加，但人工识别带宽固定**，必然走向"漏检 + 延迟"。

## 3. 目标用户与使用场景

- **一线巡检岗**：从"飞手 + 看片员"两人协作 → "无人机自飞 + 智能体自动判图 + 异常上报"
- **保护区监管岗**：从"被动接报"→"实时态势 + 风险等级 + 处置建议"主动管控
- **运营调度岗**：在出现侵限/坠物等高风险事件时，秒级联动调度，避免行车事故

典型场景：无人机或固定摄像头巡检过程中，画面中出现挖掘机臂展进入保护区、
堆土堆载超过控高、违规吊装跨越线路、人员/车辆侵入等情况，
系统需自动识别 → 判定违规等级 → 生成结构化告警单。

## 4. 量化目标（与官方 POC 验收对齐）

> 以下指标完全对齐官方赛题描述给出的"效率目标"和"POC 验证"标准。

| 指标项 | 官方目标 | 我方初赛承诺 |
|---|---|---|
| 日阅片量 | ≥ **10000 张** | 满足 |
| 综合检出率（POC 100 张混合样本） | ≥ **98%** | 优先达标方向 |
| 准确率（POC 100 张混合样本） | ≥ **90%** | 优先达标方向 |
| 吞吐速度 | 每 **100 张 ≤ 60s** | 优先达标方向 |
| 自动化程度 | 全无人自动化管控（基本取代人工判别） | 端到端流水线 |
| 应用产物形态 | 城市轨道交通保护区实施判图智能体 | Agent 化封装 |

## 5. 数据资产（官方提供 + 自建增强）

- **官方提供**：保护区不同地点的标准状态（无入侵）图像，以及各类违规侵入（如施工、堆载）的正负样本，
  典型样本数量约 **10000 张**
- **自建增强**：合成数据（违规样本 Copy-Paste / 数据增强）、公开数据集补齐（COCO 部分类别、行人/车辆/工程车类）、
  无人机视角公开数据（VisDrone 等）

## 6. 技术资源（官方提供）

- **大模型环境**：**235B QWEN3-VL**（多模态大模型，用于场景理解 / 违规规则解释 / 报告生成）
- **小模型环境**：**YOLO-V8 / V11**（实时检测器，负责高吞吐前置识别）
- **算力**：1–8 张 NVIDIA L20 或 1–8 张 昇腾 910B
- **测试地点**：**上海轨道交通全自动运行测试验证中心**
- **参与方资格**：接受 OPC 或小微型企业参与

## 7. 边界与不做范围

| 范围 | 是否纳入 |
|---|---|
| 静态图像 / 视频帧的违规识别 | ✅ 纳入（核心） |
| 多模态大模型的场景理解与违规判定解释 | ✅ 纳入（核心） |
| 自动告警单生成（结构化 JSON + 自然语言描述） | ✅ 纳入 |
| 无人机航线规划 / 飞控 | ❌ 不做（沿用既有飞控） |
| 真实保护区 GIS 红线接入 | ⚠️ 初赛用模拟红线，决赛对接 |
| 长时序事件级追踪（多帧关联） | ⚠️ 初赛单帧/短片段，决赛扩展 |

## 8. 评审对齐

针对官方四类评分维度的预备答案：

- **创新性**：YOLO 小模型（高吞吐前置）+ MLLM（语义理解 + 规则推理）+ Vision-RAG（违规规则知识库检索）三层协同
- **技术可行性**：所有组件均有成熟开源基座（YOLOv11 / Qwen2.5-VL / Milvus），可在 1–4 卡 L20 上跑通
- **业务价值**：直接对标官方"日阅片 1 万 + 取代人工"，可量化 ROI（人力替代 + 风险前置）
- **落地路径**：初赛 demo → POC 验收 → 上海全自动运行测试中心实测 → 推广至各城轨保护区
