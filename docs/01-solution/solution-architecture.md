# 解决方案架构：城市轨道交通保护区实施判图智能体

> 对应赛题：基于多模态大模型赋能的城市轨道交通保护区安全管控场景
> 应用产物：**城市轨道交通保护区实施判图智能体**
> 技术主线：**YOLO（高吞吐前置检测） + QWEN3-VL（语义理解 / 规则推理） + Vision-RAG（违规规则检索）**

---

## 1. 设计目标 → 技术映射

| 官方 KPI | 技术承载 |
|---|---|
| 日阅片量 ≥ 10000 张 | YOLO 前置检测器（CPU/GPU 批量推理）+ 异步流水线 |
| 综合检出率 ≥ 98% | YOLO 高召回阈值 + MLLM 二次复核兜底 |
| 准确率 ≥ 90% | MLLM + Vision-RAG 规则匹配剔除误检 |
| 100 张 ≤ 60s | 小模型前置 + 大模型只跑可疑帧 |
| 全无人自动化 | Agent 化封装：感知 → 判定 → 上报闭环 |

**核心思路**：**便宜的小模型先看一遍，贵的大模型只看可疑的**。
绝大多数无入侵帧由 YOLO 直接放行，只有命中候选区域的帧才送入 QWEN3-VL 做语义复核与违规判定。

## 2. 总体架构

```
┌──────────────────────────────────────────────────────────────────────┐
│                        城市轨道交通保护区判图智能体                       │
└──────────────────────────────────────────────────────────────────────┘

      [无人机/固定摄像头/车载相机]
                │ 帧序列 / 静态图
                ▼
   ┌─────────────────────────────┐
   │  L1 · 数据接入 & 预处理        │   ← 解码、抽帧、ROI 裁剪、保护区红线掩码
   └─────────────────────────────┘
                │
                ▼
   ┌─────────────────────────────┐
   │  L2 · YOLO 前置检测           │   ← YOLOv8/v11 多任务头
   │     · 12 类违规目标框          │      (人/工程车/挖掘机/吊车/堆土/管材/
   │     · 高召回阈值（保检出率）    │       异物/施工设备/违建/围挡/电缆/动物)
   └─────────────────────────────┘
                │ 候选帧 + bbox + 类别
                ▼  ┌─── 无候选 → 直接放行（计入"已审核未发现"）
   ┌─────────────────────────────┐
   │  L3 · MLLM 语义复核（QWEN3-VL）│   ← 多模态大模型读图 + 读规则
   │     · 是否真违规？             │
   │     · 违规等级（轻/中/重/紧急） │
   │     · 自然语言违规描述          │
   └─────────────────────────────┘
                │
                ▼
   ┌─────────────────────────────┐
   │  L4 · Vision-RAG 规则检索      │   ← 检索"保护区管理办法"知识库
   │     · 命中条款编号 + 原文        │      (Milvus + BGE-M3 + 图文向量)
   └─────────────────────────────┘
                │
                ▼
   ┌─────────────────────────────┐
   │  L5 · 告警单生成 & 推送         │   ← 结构化 JSON + 自然语言报告
   │     · 等级 / 位置 / 证据图      │     可对接调度系统 / 巡检平台
   └─────────────────────────────┘
                │
                ▼
        [运营平台 / 调度大屏 / 微信告警]
```

## 3. 12 类典型违规目标（YOLO 检出类）

| # | 类别 | 风险等级 | 备注 |
|---|---|---|---|
| 1 | 挖掘机臂展进入保护区 | 紧急 | 直接威胁线路 |
| 2 | 吊车 / 塔吊跨越线路 | 紧急 | 坠物高风险 |
| 3 | 堆土超高 | 重 | 雨季滑塌风险 |
| 4 | 违章建筑 | 中 | 长期占压 |
| 5 | 违规堆载（管材/钢筋） | 中 | 占压保护区 |
| 6 | 违规倾倒渣土 | 中 | 累积坍塌风险 |
| 7 | 工程车 / 施工车辆 | 中 | 需结合是否违规进入 |
| 8 | 人员侵入限界 | 重 | 行车安全 |
| 9 | 异物坠落 / 大件杂物 | 重 | 含气球、塑料布、风筝线 |
| 10 | 临时围挡侵入 | 中 | 占用作业带 |
| 11 | 高压电缆 / 通信线路侵入 | 重 | 电力安全 |
| 12 | 大型动物侵入 | 中 | 偶发但需上报 |

> 12 类清单为初赛优先实现集合，最终类目以官方数据集为准。

## 4. 关键模块设计

### 4.1 YOLO 前置检测（L2）

- **基座**：YOLOv11-l（精度优先）/ YOLOv8-n（速度优先），按算力卡数自适应切换
- **训练策略**：官方 ~10000 张 + Mosaic / Copy-Paste 增强 + 难样本回灌
- **阈值策略**：低 conf 高 IoU，**优先保召回（≥ 99% 候选召回）**
- **输出**：(frame_id, bbox, class_id, conf) 列表

### 4.2 MLLM 语义复核（L3）

- **基座**：官方 235B QWEN3-VL（线上）/ Qwen2.5-VL-7B（本地兜底）
- **输入**：原图 + YOLO bbox crop + 候选类别 + 检索到的相关规则
- **Prompt 结构**：

  ```text
  你是城市轨道交通保护区安全管控专家。
  以下图像位于保护区 [{location}]，YOLO 初步识别到 [{candidates}]。
  请根据保护区管理规定（见参考条款），判断：
  1. 是否构成违规？(是/否)
  2. 违规等级？(轻/中/重/紧急)
  3. 违规描述（一句话）
  4. 命中条款编号
  以 JSON 输出。
  参考条款：{rag_results}
  ```

- **输出**：结构化 JSON `{is_violation, level, desc, clause_ids}`

### 4.3 Vision-RAG 规则知识库（L4）

- **向量库**：Milvus / Qdrant
- **Embedding**：BGE-M3（文本）+ CLIP / EVA-CLIP（图像）
- **入库内容**：保护区管理办法、违规判定细则、典型案例图文对（图 + 描述 + 条款）
- **检索策略**：Hybrid（文本 + 图像）→ Rerank → Top-K 5 条注入 Prompt

### 4.4 Agent 编排（L5）

- **框架**：LangGraph / Dify / 自研轻量 FSM
- **工具集**：YOLO 推理工具、MLLM 复核工具、RAG 检索工具、告警单生成工具、运营平台推送工具
- **状态机**：接帧 → 检测 → 路由（放行/复核） → 复核 → 检索 → 生成告警 → 推送 → 闭环

## 5. 技术选型表

| 层 | 选型 | 备选 | 理由 |
|---|---|---|---|
| 检测器 | YOLOv11-l | YOLOv8-n / RT-DETR | 官方明确 v8/v11 环境 |
| MLLM | QWEN3-VL-235B | Qwen2.5-VL-7B / InternVL2 | 官方提供环境 |
| Embedding（文） | BGE-M3 | bge-large-zh | 中文/多语强 |
| Embedding（图） | EVA-CLIP-L | OpenCLIP | 视觉细粒度 |
| 向量库 | Milvus 2.x | Qdrant | 开源、横向扩展 |
| Agent | LangGraph | Dify / 自研 | 状态可控 |
| 推理服务 | vLLM + Triton | TensorRT-LLM | 兼容多后端 |
| 部署 | Docker Compose（初赛）→ K8s（决赛） | — | 演示友好 |

## 6. 性能预算（针对 100 张 ≤ 60s POC）

| 阶段 | 单图耗时（目标） | 备注 |
|---|---|---|
| 解码 + 预处理 | 30 ms | CPU |
| YOLO 推理 | 80 ms | L20 单卡 batch=8 |
| 路由判断 | 5 ms | — |
| MLLM 复核（命中候选才跑） | 1.5 s × 命中率 30% = 450 ms | 等价分摊 |
| RAG 检索 | 50 ms | Milvus + GPU rerank |
| 告警生成 | 20 ms | 模板拼接 |
| **合计 / 张** | **~ 600 ms** | 留 100 ms 余量 |

100 张 × 600 ms = 60s，**贴线达标**；通过批处理 + 并发可进一步提速。

## 7. 端云部署方案

- **云端**：QWEN3-VL-235B + Milvus + 编排服务（GPU 集群）
- **边缘**：无人机 / 巡检车上的 Jetson AGX Orin / 昇腾 Atlas 跑 YOLOv11-n + 关键帧上传
- **断网兜底**：边缘端缓存 + 重连后回传

## 9. 风险与缓解

| 风险 | 缓解 |
|---|---|
| 数据稀缺（违规样本少） | Copy-Paste / 合成数据 / 公开数据预训练 |
| MLLM 推理慢拖累吞吐 | 路由策略 + batch + 仅复核可疑帧 |
| 误检率高 | RAG 引入条款约束 + 阈值分级 |
| 算力不稳定 | 边缘小模型兜底 + 关键帧才上云 |
| 多类目长尾 | 难类样本回灌 + 主动学习 |
