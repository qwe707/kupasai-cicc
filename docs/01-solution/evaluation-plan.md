# 评估方案

## 一、目标
量化系统在「检索 / 生成 / 落地」三层的表现，作为方案说明书与决赛报告的硬证据。

## 二、评估维度

| 层 | 指标 | 测量方式 | 目标 |
| --- | --- | --- | --- |
| 检索 | Top-1 / Top-5 命中率 | 自建评估集 | Top-5 ≥ 90% |
| 检索 | MRR | 自建评估集 | ≥ 0.75 |
| 生成 | 引用准确率 | 人工标注 | ≥ 95% |
| 生成 | 答案有用性（1-5） | 人工评分 | ≥ 4 |
| 系统 | P50 响应时间 | 压测 | ≤ 3s |
| 系统 | P95 响应时间 | 压测 | ≤ 6s |
| 落地 | 用户满意度 | 试用反馈 | ≥ 4/5 |

## 三、评估集

- 至少 200 道题，覆盖：
  - 简单条款查询
  - 跨章节关联
  - 跨规范冲突
  - 强条识别
  - 数值计算辅助
- 每题包含：问题、参考答案、引用条款编号、难度等级

## 四、对比基线

| 基线 | 说明 |
| --- | --- |
| 关键词检索 | BM25 only |
| 通用大模型 | 不接 RAG |
| 通用 RAG | 普通向量检索 + LLM |
| 本方案 | 混合检索 + 强引用 + 规则引擎 |

## 五、报告产出

- `src/evaluation/` 下放评估脚本
- 结果输出到 `deliverables/{round}/evaluation-report.md`
- 关键图表导出到 `assets/posters/`
